# Security Policy

## Supported versions
Main branch only until formal releases begin.

## Reporting vulnerabilities
- Open a **Security** issue with minimal reproducible details,  
  or email maintainers (see README).  
- Do not post sensitive data; use synthetic samples.  
- Acknowledge within 72 hours · Patch promptly.

## Scope
- Data handling and audit integrity  
- Containment rules and policy gates  
- RMRIΔ calculations affecting regulation behaviour

## Guidance
- Default-deny on uncertainty.  
- Prefer redaction to rejection.  
- Every regulation decision must be auditable (timestamp + rationale).
